# Customer Archive Rules

The archive doc is a decision aid, not a raw dump, not a table export, and not a running log.

## Uniqueness rule

- Each customer should have exactly one canonical customer archive doc.
- If a new history summary, supplement, or analysis note is needed, merge it into the existing archive doc instead of creating a second archive.
- Only create a new archive doc when the customer has no archive doc yet.

## Recommended sections

- Basic information
- Latest public updates
- Cooperation history
- Org and key contacts
- System or project status
- Competitor situation
- Visit and communication records
- Current account judgment
- Current action items

## Position in the workbench

- The archive is the customer's full-picture document.
- It should explain the customer's historical arc, current reality, key people, major risks, opportunities, and why the current operating strategy makes sense.
- It is allowed to synthesize from detail tables, Todo items, local files, and public updates.
- It should become the basis for later strategy changes in `客户主数据`.
- It should not simply mirror every detail row from Base.

## Section rules

### Latest public updates

- Only public information belongs here.
- Good sources: earnings releases, investor materials, leadership interviews, public org changes, industry reports, funding or strategy news.
- Each item should include source, date, and why it matters.
- Do not place internal meetings or private notes here.

### Cooperation history

- Include historical project scope, delivered value, money facts, contract periods, and payment status.
- Distinguish actual contracts from historical plans or optional fee clauses.
- Prefer stage-based narration over a raw chronological dump when that makes the account easier to understand.

### Org and key contacts

- Separate current contacts from historical contacts.
- Historical contacts can stay for context, but they should not be shown as current decision chain members.

### Visit and communication records

- Use for meetings, calls, visits, and direct follow-up notes.
- Include date, participants, summary, and agreed next steps.
- Do not paste full transcripts into the archive if a dedicated meeting-note doc exists. Link the cold-memory doc instead.
- Keep this section selective. The archive should show the important communication thread, not every minor touchpoint.

### Current account judgment

- Put strategy, risks, opportunities, and recommended posture here.
- This section may include informed judgment, but it must be clearly judgment rather than objective fact.
- If the judgment has matured enough to justify a change in `客户主数据`, say so explicitly in the change plan.

## Editing rule

If the current archive template conflicts with these rules, prefer these rules and note the mismatch in the response.
